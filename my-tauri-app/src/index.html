🦀 Tauri Feature Explorer

Learn how Tauri works by interacting with each concept below

1

Commands

Concept: The #[tauri::command] macro turns a Rust function into something you can call from JavaScript via invoke("command_name", { args }). Arguments are auto-deserialized and return values auto-serialized.

Show Rust code 
#[tauri::command]
fn greet(name: &str) -> String {
    format!("Hello, {}! Greeted from Rust!", name)
}

// JS side:
// await invoke("greet", { name: "World" })
2

System Info from Rust

Concept: Commands can return complex structs. Tauri uses serde to serialize Rust structs into JSON automatically. The JS side receives a plain JavaScript object.

Show Rust code 
#[derive(Serialize)]
struct SystemInfo {
    os_name: String,
    arch: String,
    num_cpus: usize,
}

#[tauri::command]
fn get_system_info() -> SystemInfo { ... }
3

State Management

Concept: Tauri manages shared state on the Rust side using .manage(AppState { ... }). Commands inject it via State<'_, AppState>. Wrap mutable data in Mutex for thread safety. State persists for the app's entire lifetime.

0

This counter lives in Rust memory (Mutex<i32>), not in JavaScript.

Show Rust code 
struct AppState {
    counter: Mutex<i32>,
}

#[tauri::command]
fn update_counter(delta: i32, state: State<'_, AppState>) -> i32 {
    let mut counter = state.counter.lock().unwrap();
    *counter += delta;
    *counter
}
4

Complex Data (Notes CRUD)

Concept: Tauri seamlessly passes complex data between JS and Rust. Structs, Vecs, nested objects — all auto-serialized via serde. Here we manage a Vec<Note> in Rust state.

Show Rust code 
#[derive(Serialize, Deserialize, Clone)]
struct Note { id: usize, text: String }

#[tauri::command]
fn add_note(text: String, state: State<'_, AppState>) -> Vec<Note> {
    let mut notes = state.notes.lock().unwrap();
    notes.push(Note { id, text });
    notes.clone()
}
5

File I/O + Events

Concept: Rust can read/write files using std::fs. After operations, Rust can emit events to the frontend via app.emit("event-name", payload). The JS side listens with listen("event-name", callback). This is a push mechanism — Rust notifies JS.

6

Error Handling

Concept: Commands can return Result<T, String>. Ok(value) resolves the JS promise. Err(msg) rejects it — caught with .catch() or try/catch in async/await.

÷

Try dividing by 0 to see Rust send an error!

Show Rust code 
#[tauri::command]
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err("Cannot divide by zero!".to_string())
    } else {
        Ok(a / b)
    }
}
